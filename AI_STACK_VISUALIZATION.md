// 질문 1: "여행 계획을 세울 때 당신은?"

Question {
  id: 1,
  options: [
    {
      value: "detailed_plan",
      scores: {
        plan: 2,        // ← 계획성 +2
        safe: 1         // ← 안전성 +1
      }
    }
  ]
}

// 12문항 완료 후 집계 결과
scores = {
  plan: 8,
  spontaneous: 2,
  adventure: 3,
  safe: 7,
  active: 4,
  rest: 6,
  social: 3,
  solo: 5,
  healing: 7,
  aesthetic: 4,
  food: 2,
  nature: 5,
  easygoing: 1
}

// 축별 점수
planScore = 8 - 2 = 6          (계획적)
adventureScore = 3 - 7 = -4    (안전추구)
activeScore = 4 - 6 = -2       (휴식선호)

// 판별 결과: Master Planner (계획적 + 안전추구)

### 2.2 백엔드 성향 분석 (Python)

#### 🎯 목적
사용자 행동 데이터를 기반으로 성향을 동적으로 학습하고 업데이트

#### 📊 알고리즘 플로우차트

```
[초기화] 온보딩 답변 입력
   │
   ▼
┌──────────────────────────────────────────┐
│  initialize_preference()                 │
├──────────────────────────────────────────┤
│  1. 답변 → 태그 가중치 매핑              │
│     "힐링" 선택 → {"힐링": 0.5}          │
│     "자연" 선택 → {"자연": 0.3}          │
│                                          │
│  2. 활동 수준 조정                       │
│     IF activity="high" → 액티비티+       │
│     IF activity="low" → 힐링+            │
│                                          │
│  3. 벡터 정규화 (L1 Norm)                │
│     vector = profile / sum(profile)      │
└──────────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────────┐
│  PreferenceProfile 생성                  │
├──────────────────────────────────────────┤
│  {                                       │
│    profile: {"힐링": 0.7, "액티비티": 0.3}│
│    vector: [0.7, 0.3],                   │
│    weights: {"힐링": 1.4, "액티비티": 0.6}│
│  }                                       │
└──────────────────────────────────────────┘
   │
   ▼
[사용자 행동 이벤트 발생]
   │
   ▼
┌──────────────────────────────────────────┐
│  update_preference_from_event()          │
├──────────────────────────────────────────┤
│  Event: {                                │
│    type: "click",                        │
│    tags: ["카페", "감성"],               │
│  }                                       │
│                                          │
│  1. 이벤트 가중치 설정                   │
│     click: 0.5                           │
│     view: 0.3                            │
│     purchase: 1.0                        │
│                                          │
│  2. 학습률 적용 업데이트                 │
│     α = 0.1 (학습률)                     │
│     preference[tag] +=                   │
│       α × event_weight × tag_base_weight │
│                                          │
│  3. 재정규화                             │
└──────────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────────┐
│  detect_preference_change()              │
├──────────────────────────────────────────┤
│  1. 코사인 유사도 계산                   │
│     similarity = (v1·v2)/(||v1||×||v2||) │
│                                          │
│  2. 변화 감지                            │
│     IF (1 - similarity) > 0.1:           │
│       → 성향 변화 발생!                  │
└──────────────────────────────────────────┘
```


#### 🔢 수학적 공식

##### 1. 벡터 정규화 (L1 Normalization)

```
profile = {"힐링": 1.2, "액티비티": 0.6, "맛집": 0.4}

총합: sum = 1.2 + 0.6 + 0.4 = 2.2

정규화:
vector[0] = 1.2 / 2.2 = 0.545  (힐링)
vector[1] = 0.6 / 2.2 = 0.273  (액티비티)
vector[2] = 0.4 / 2.2 = 0.182  (맛집)

결과: [0.545, 0.273, 0.182]
총합 = 1.0 (정규화 완료)
```

##### 2. 학습률 기반 업데이트

```
현재 성향: preference["감성"] = 0.4

이벤트: 사용자가 "감성 카페" 클릭
- event_type: "click" → weight = 0.5
- tag: "감성" → tag_base_weight = 0.25
- 학습률: α = 0.1

업데이트:
preference["감성"] = 0.4 + (0.1 × 0.5 × 0.25)
                  = 0.4 + 0.0125
                  = 0.4125

이후 전체 프로파일 재정규화
```

##### 3. 코사인 유사도 (Cosine Similarity)

```
이전 벡터: v1 = [0.7, 0.2, 0.1]  (힐링 중심)
현재 벡터: v2 = [0.5, 0.3, 0.2]  (액티비티 증가)

1. 내적 (Dot Product)
   v1 · v2 = (0.7×0.5) + (0.2×0.3) + (0.1×0.2)
          = 0.35 + 0.06 + 0.02
          = 0.43

2. 벡터 크기 (Norm)
   ||v1|| = √(0.7² + 0.2² + 0.1²) = √0.54 = 0.735
   ||v2|| = √(0.5² + 0.3² + 0.2²) = √0.38 = 0.616

3. 코사인 유사도
   similarity = 0.43 / (0.735 × 0.616)
              = 0.43 / 0.453
              = 0.949

4. 변화 판정
   change = 1 - 0.949 = 0.051 < 0.1
   → 변화 없음 (임계값 0.1)
```

#### 📈 성향 변화 추적 예시

```
시간 흐름에 따른 성향 벡터 변화

Week 1 (초기):
  [힐링: 0.7, 액티비티: 0.2, 맛집: 0.1]
  주요 성향: 휴식 중심

Week 4 (클릭/조회 이벤트 100회):
  [힐링: 0.6, 액티비티: 0.3, 맛집: 0.1]
  similarity = 0.95 → 변화 없음

Week 8 (액티비티 상품 구매 5회):
  [힐링: 0.5, 액티비티: 0.4, 맛집: 0.1]
  similarity = 0.88 → 변화 감지! ⚠️

Week 12 (성향 전환 완료):
  [힐링: 0.3, 액티비티: 0.5, 맛집: 0.2]
  주요 성향: 액티비티 중심

→ 추천 알고리즘 재조정 트리거
```

## 3. 일정 생성 AI

### 3.1 전체 프로세스

```
[입력] 추천 장소 10개 + 여행 일수 3일
   │
   ▼
┌──────────────────────────────────────────┐
│  Generator.generate()                    │
│  (일정 생성 메인 로직)                   │
└──────────────────────────────────────────┘
   │
   ├─ Step 1: 지리적 클러스터링
   │    │
   │    ▼
   │  ┌────────────────────────────────┐
   │  │ _cluster_items()               │
   │  ├────────────────────────────────┤
   │  │ 도시별로 장소 그룹화:          │
   │  │ • 부산 시내: 5개               │
   │  │ • 부산 해운대: 3개             │
   │  │ • 부산 외곽: 2개               │
   │  └────────────────────────────────┘
   │
   ├─ Step 2: 날짜별 분배
   │    │
   │    ▼
   │  ┌────────────────────────────────┐
   │  │ _distribute_across_days()      │
   │  ├────────────────────────────────┤
   │  │ Round-robin 방식:              │
   │  │ • Day 1: 3개 (클러스터 A)      │
   │  │ • Day 2: 4개 (클러스터 B)      │
   │  │ • Day 3: 3개 (클러스터 C)      │
   │  └────────────────────────────────┘
   │
   ├─ Step 3: 동선 최적화 (각 날짜별)
   │    │
   │    ▼
   │  ┌────────────────────────────────┐
   │  │ Optimizer.optimize_route()     │
   │  ├────────────────────────────────┤
   │  │ TSP 근사 알고리즘:             │
   │  │ [A → B → C] → [A → C → B]      │
   │  │ 거리: 15km → 10km (최적화)     │
   │  └────────────────────────────────┘
   │
   └─ Step 4: 시간 슬롯 배치
        │
        ▼
      ┌────────────────────────────────┐
      │ Scheduler.schedule_items()     │
      ├────────────────────────────────┤
      │ 성향 기반 일정 밀도:           │
      │ • 힐링형: 3-4개/일             │
      │ • 액티비티형: 6-7개/일         │
      └────────────────────────────────┘
        │
        ▼
[출력] 최적화된 3일 일정 완성
```
