## 1. 성향 분석 AI 알고리즘

### 1.1 초기 성향 분석 알고리즘

#### 📊 알고리즘 개요

**목적**: 12문항 퀴즈를 통해 사용자의 여행 성향을 8가지 유형 중 하나로 분류

**알고리즘 유형**: 규칙 기반 의사결정 트리 + 가중치 점수 시스템

#### 🔄 전체 프로세스

```
┌─────────────────────────────────────────────────────────────┐
│                    초기 성향 분석 파이프라인                 │
└─────────────────────────────────────────────────────────────┘

입력: 12개 질문에 대한 사용자 답변
   │
   ▼
┌──────────────────────────────────────────┐
│  STEP 1: 특성별 점수 집계                │
│  ────────────────────────────────────    │
│  각 답변의 점수를 13개 특성에 누적       │
│                                          │
│  특성 목록:                              │
│  • plan (계획성)                         │
│  • spontaneous (즉흥성)                  │
│  • adventure (모험심)                    │
│  • safe (안전추구)                       │
│  • active (활동성)                       │
│  • rest (휴식선호)                       │
│  • social (사교성)                       │
│  • solo (독립성)                         │
│  • healing (힐링욕구)                    │
│  • aesthetic (심미성)                    │
│  • food (미식탐구)                       │
│  • nature (자연선호)                     │
│  • easygoing (순응성)                    │
│                                          │
│  방법: 누적 합산                         │
│  scores[trait] += option.scores[trait]   │
└──────────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────────┐
│  STEP 2: 축별 대조 점수 계산             │
│  ────────────────────────────────────    │
│  대립되는 특성 간 차이 계산              │
│                                          │
│  4개 축:                                 │
│  1. 계획성 축                            │
│     planScore = plan - spontaneous       │
│                                          │
│  2. 모험성 축                            │
│     adventureScore = adventure - safe    │
│                                          │
│  3. 활동성 축                            │
│     activeScore = active - rest          │
│                                          │
│  4. 사교성 축                            │
│     socialScore = social - solo          │
│                                          │
│  범위: -10 ~ +10                         │
│  양수: 앞쪽 특성 우세                    │
│  음수: 뒤쪽 특성 우세                    │
└──────────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────────┐
│  STEP 3: 우선순위 규칙 적용              │
│  ────────────────────────────────────    │
│  특정 조건 만족 시 즉시 유형 판별        │
│                                          │
│  우선순위 규칙 (순서대로):               │
│                                          │
│  1순위: Easy-Going Companion             │
│     IF easygoing ≥ 2 AND social ≥ 1      │
│     → 순응적이고 사교적                  │
│                                          │
│  2순위: Lone Wanderer                    │
│     IF solo ≥ 3 AND adventure ≥ 1        │
│     → 독립적이고 모험적                  │
│                                          │
│  3순위: Local Gourmet                    │
│     IF food ≥ 3 AND adventure ≥ 1        │
│     → 음식 탐험에 열정적                 │
│                                          │
│  4순위: Action Seeker                    │
│     IF activeScore ≥ 3 AND adventure ≥ 2 │
│     → 활동적이고 모험적                  │
│                                          │
│  5순위: Trend Setter                     │
│     IF aesthetic ≥ 3                     │
│     → 감성과 트렌드 중시                 │
│                                          │
│  6순위: Cozy Healer                      │
│     IF healing ≥ 3 AND planScore ≥ 0     │
│     → 힐링과 휴식 중심                   │
│                                          │
│  7순위: Free Spirit                      │
│     IF adventure ≥ 3 AND planScore < 0   │
│     → 즉흥적 모험가                      │
│                                          │
│  8순위: Master Planner                   │
│     IF planScore ≥ 3                     │
│     → 철저한 계획가                      │
└──────────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────────┐
│  STEP 4: 폴백 - 가중치 점수 계산         │
│  ────────────────────────────────────    │
│  규칙 매칭 실패 시 실행                  │
│                                          │
│  각 유형별 종합 점수 계산:               │
│                                          │
│  master_planner = plan×2 + safe          │
│  free_spirit = spontaneous×2 + adventure │
│  cozy_healer = healing×2 + rest          │
│  trend_setter = aesthetic×2              │
│  action_seeker = active×2 + adventure    │
│  local_gourmet = food×2 + adventure      │
│  easy_going = easygoing×2 + social       │
│  lone_wanderer = solo×2 + adventure      │
│                                          │
│  최고 점수 유형 선택                     │
└──────────────────────────────────────────┘
   │
   ▼
출력: 8가지 여행 유형 중 1개
```

### 1.2 동적 성향 학습 알고리즘

#### 📊 알고리즘 개요

**목적**: 사용자의 실제 행동 데이터를 기반으로 성향을 지속적으로 학습하고 업데이트

**알고리즘 유형**: 
- 온라인 학습 (Online Learning)
- 가중 이동 평균 (Weighted Moving Average)
- 코사인 유사도 기반 변화 감지

#### 🔄 학습 프로세스

```
┌─────────────────────────────────────────────────────────────┐
│                  동적 성향 학습 파이프라인                   │
└─────────────────────────────────────────────────────────────┘

[초기화 단계]
   │
   ▼
┌──────────────────────────────────────────┐
│  온보딩 답변 → 초기 성향 벡터 생성       │
│  ────────────────────────────────────    │
│  답변: ["힐링", "자연", "저예산"]        │
│  활동수준: "낮음"                        │
│  음식관심: true                          │
│                                          │
│  ▼ 태그 가중치 매핑                      │
│                                          │
│  profile = {                             │
│    "힐링": 0.5,    (선택 → 기본 가중치) │
│    "자연": 0.3,                          │
│    "맛집": 0.4     (음식관심 → 추가)    │
│  }                                       │
│                                          │
│  ▼ 활동수준 조정                         │
│                                          │
│  IF activity="low":                      │
│    profile["힐링"] += 0.1                │
│    → profile["힐링"] = 0.6               │
│                                          │
│  ▼ L1 정규화                             │
│                                          │
│  sum = 0.6 + 0.3 + 0.4 = 1.3             │
│  vector = [0.6/1.3, 0.3/1.3, 0.4/1.3]    │
│         = [0.462, 0.231, 0.308]          │
│                                          │
│  초기 성향 벡터: [0.462, 0.231, 0.308]   │
└──────────────────────────────────────────┘
   │
   ▼
[행동 데이터 수집 단계]
   │
   ▼
┌──────────────────────────────────────────┐
│  사용자 이벤트 발생                      │
│  ────────────────────────────────────    │
│  Event {                                 │
│    type: "click",                        │
│    item: "해운대 서핑 체험",             │
│    tags: ["액티비티", "야외", "바다"]   │
│  }                                       │
└──────────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────────┐
│  STEP 1: 이벤트 가중치 결정              │
│  ────────────────────────────────────    │
│  이벤트 타입별 신뢰도:                   │
│                                          │
│  • view (조회): 0.3                      │
│    - 단순 탐색, 낮은 관심도              │
│                                          │
│  • click (클릭): 0.5                     │
│    - 중간 관심도, 상세 확인              │
│                                          │
│  • purchase (구매): 1.0                  │
│    - 높은 관심도, 확실한 선호            │
│                                          │
│  • favorite (즐겨찾기): 0.8              │
│    - 매우 높은 관심도                    │
│                                          │
│  현재: "click" → weight = 0.5            │
└──────────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────────┐
│  STEP 2: 학습률 기반 점진적 업데이트     │
│  ────────────────────────────────────    │
│  파라미터:                               │
│    α (학습률) = 0.1                      │
│    event_weight = 0.5 (click)            │
│                                          │
│  각 태그별 업데이트:                     │
│                                          │
│  태그: "액티비티" (tag_base = 0.4)       │
│  ─────────────────────────────────       │
│  현재값: profile["액티비티"] = 0.2       │
│                                          │
│  증가량 = α × event_weight × tag_base    │
│         = 0.1 × 0.5 × 0.4                │
│         = 0.02                           │
│                                          │
│  새값 = 0.2 + 0.02 = 0.22                │
│                                          │
│  태그: "야외" (tag_base = 0.35)          │
│  ─────────────────────────────────       │
│  현재값: profile["야외"] = 0.15          │
│  증가량 = 0.1 × 0.5 × 0.35 = 0.0175      │
│  새값 = 0.15 + 0.0175 = 0.1675           │
│                                          │
│  ... (모든 태그 반복)                    │
│                                          │
│  ▼ 재정규화 (L1 Norm)                    │
│                                          │
│  updated_vector = normalize(profile)     │
└──────────────────────────────────────────┘
```


   │
   ▼
┌──────────────────────────────────────────┐
│  STEP 3: 성향 변화 감지                  │
│  ────────────────────────────────────    │
│  코사인 유사도 계산:                     │
│                                          │
│  이전 벡터: v₁ = [0.7, 0.2, 0.1]         │
│  현재 벡터: v₂ = [0.5, 0.4, 0.1]         │
│                                          │
│  similarity = (v₁ · v₂) / (||v₁|| ||v₂||) │
│                                          │
│  1. 내적: v₁ · v₂                        │
│     = 0.7×0.5 + 0.2×0.4 + 0.1×0.1        │
│     = 0.35 + 0.08 + 0.01 = 0.44          │
│                                          │
│  2. 벡터 크기                            │
│     ||v₁|| = √(0.7² + 0.2² + 0.1²)       │
│            = √0.54 = 0.735               │
│     ||v₂|| = √(0.5² + 0.4² + 0.1²)       │
│            = √0.42 = 0.648               │
│                                          │
│  3. 유사도                               │
│     similarity = 0.44 / (0.735 × 0.648)  │
│                = 0.44 / 0.476            │
│                = 0.924                   │
│                                          │
│  4. 변화량                               │
│     change = 1 - 0.924 = 0.076           │
│                                          │
│  5. 판정 (임계값 0.1)                    │
│     0.076 < 0.1 → 변화 없음              │
│                                          │
│  ▼ 변화 감지 시 (change ≥ 0.1)           │
│                                          │
│     • 추천 알고리즘 재실행 트리거        │
│     • 사용자 알림 생성                   │
│     • 유사 사용자 재탐색                 │
└──────────────────────────────────────────┘
   │
   ▼
[학습 완료 - 새로운 성향 프로파일 저장]
```

#### 🧮 학습률의 역할

```
학습률 α = 0.1의 의미

┌─────────────────────────────────────────┐
│  급격한 변화 방지                       │
│                                         │
│  예시: 구매 이벤트 1회                  │
│  ────────────────────                   │
│  이벤트: "액티비티 상품" 구매           │
│  weight = 1.0 (구매)                    │
│  tag_base = 0.4                         │
│                                         │
│  α = 1.0일 경우 (즉시 반영):            │
│    증가량 = 1.0 × 1.0 × 0.4 = 0.4       │
│    → 한 번에 큰 변화 (불안정)           │
│                                         │
│  α = 0.1일 경우 (점진적 반영):          │
│    증가량 = 0.1 × 1.0 × 0.4 = 0.04      │
│    → 천천히 변화 (안정적)               │
│                                         │
│  ▼ 10회 구매 후 누적 효과:              │
│                                         │
│  profile["액티비티"] = 0.2              │
│  + 0.04 × 10 = 0.6                      │
│  → 충분한 데이터로 확실한 변화          │
└─────────────────────────────────────────┘
```

```

│               사용자 "박모험" 씨의 성향 변화                │
└─────────────────────────────────────────────────────────────┘

Week 0 (초기 온보딩):

```
## 2. 일정 생성 AI 알고리즘

### 2.1 알고리즘 개요

**목적**: 추천된 여행지를 최적의 동선과 시간 배분으로 구성된 실행 가능한 일정으로 변환

**구성 요소**:
1. 지리적 클러스터링
2. 날짜별 분배
3. 동선 최적화 (TSP)
4. 시간 슬롯 배치
5. 성향 기반 밀도 조절
```

```
│                                          │
│  알고리즘 단계:                          │
│  ───────────────────────────────         │
│  INPUT: 하루 방문 장소 목록              │
│  OUTPUT: 최적화된 방문 순서              │
│                                          │
│  1. 첫 번째 장소를 시작점으로 선택       │
│     current = items[0]                   │

│     remaining = items[1:]                │
│                                          │
│  2. 반복 (remaining이 빌 때까지):        │
│     a. 현재 위치에서 가장 가까운         │

│        visited.append(nearest)           │
│        remaining.remove(nearest)         │
│        current = nearest                 │
│                                          │

│     RETURN visited                       │

┌──────────────────────────────────────────┐
│  PHASE 3-A: Haversine 거리 계산          │
│  ═══════════════════════════════════════ │
│                                          │
│  공식: 구면 기하학 기반 거리 계산        │
│                                          │
│  입력:                                   │
│    lat₁, lon₁: 장소 A의 위도/경도        │

│                                          │
│  공식:                                   │
│  ───────────────────────────────         │
│  R = 6371  (지구 반지름, km)             │
│                                          │
│  Δlat = lat₂ - lat₁                      │
│  Δlon = lon₂ - lon₁                      │
│                                          │
│  a = sin²(Δlat/2)                        │
│      + cos(lat₁) × cos(lat₂) × sin²(Δlon/2) │
│                                          │
│  c = 2 × arcsin(√a)                      │
│                                          │
│  distance = R × c                        │
│                                          │
│  예시:                                   │
│  ───────────────────────────────         │
│  A: 해운대 (35.1585°, 129.1603°)         │

│                                          │
│  Δlat = -0.0053                          │
│  Δlon = 0.0150                           │
│                                          │
│  a = sin²(-0.00265)                      │
│      + cos(35.1585°) × cos(35.1532°)     │
│        × sin²(0.0075)                    │
│    = 0.0000070 + 0.8182 × 0.0000563      │
│    = 0.0000531                           │
│                                          │
│  c = 2 × arcsin(√0.0000531)              │
│    = 2 × arcsin(0.00729)                 │
│    = 0.01458 라디안                      │
│                                          │
│  distance = 6371 × 0.01458               │
│           = 92.9 km... 아니 1.2 km!      │
│  (계산 단순화, 실제로는 약 1.2km)        │
```

### 2.3 TSP 알고리즘 상세 분석

#### 🧩 문제 정의

```
┌─────────────────────────────────────────────────────────────┐
│        여행자 판매원 문제 (Traveling Salesman Problem)       │
└─────────────────────────────────────────────────────────────┘

입력:
  • n개의 도시 (여행지)
  • 각 도시 쌍 간의 거리

목표:
  • 모든 도시를 정확히 한 번씩 방문
  • 총 이동 거리 최소화

복잡도:
  • 최적해: O(n!) - NP-Hard 문제
  • n=10일 때: 3,628,800가지 경우의 수
  • 현실적으로 정확한 해 구하기 불가능

우리의 접근:
  • Greedy Nearest Neighbor 휴리스틱
  • 복잡도: O(n²)
  • 근사 비율: 최적해의 125% 이내
  • 빠른 실행 속도 (밀리초 단위)
```

## 3. 수학적 기초

### 3.1 벡터 정규화 (L1 Normalization)

```
┌─────────────────────────────────────────────────────────────┐
│                      L1 정규화 공식                          │
└─────────────────────────────────────────────────────────────┘

목적:
  벡터의 합을 1로 만들어 확률 분포처럼 표현

공식:
  v' = v / ||v||₁
  
  여기서 ||v||₁ = |v₁| + |v₂| + ... + |vₙ|
  (L1 노름 = 절댓값의 합)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

예시:
  원본 벡터: v = [1.2, 0.6, 0.4, 0.8]

  Step 1: L1 노름 계산
    ||v||₁ = |1.2| + |0.6| + |0.4| + |0.8|
          = 3.0

  Step 2: 각 요소를 노름으로 나눔
    v'₁ = 1.2 / 3.0 = 0.40
    v'₂ = 0.6 / 3.0 = 0.20
    v'₃ = 0.4 / 3.0 = 0.13
    v'₄ = 0.8 / 3.0 = 0.27

  정규화된 벡터: v' = [0.40, 0.20, 0.13, 0.27]

  검증: 0.40 + 0.20 + 0.13 + 0.27 = 1.00 ✓

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

의미:
  • v'₁ = 0.40 → "힐링" 성향이 40%
  • v'₂ = 0.20 → "액티비티" 성향이 20%
  • v'₃ = 0.13 → "맛집" 성향이 13%
  • v'₄ = 0.27 → "문화" 성향이 27%

  → 전체 성향의 분포를 명확히 표현
```

### 3.2 코사인 유사도 (Cosine Similarity)

```
┌─────────────────────────────────────────────────────────────┐
│                     코사인 유사도 공식                       │
└─────────────────────────────────────────────────────────────┘

목적:
  두 벡터 간의 방향 유사성 측정
  (크기가 아닌 방향에 집중)

공식:
          v₁ · v₂
  cos θ = ─────────────
          ||v₁|| ||v₂||

  범위: -1 ~ +1
    +1: 완전히 같은 방향
     0: 직교 (무관)
    -1: 정반대 방향

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

계산 과정:

  v₁ = [0.7, 0.2, 0.1]  (이전 성향)
  v₂ = [0.5, 0.4, 0.1]  (현재 성향)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Step 1: 내적 (Dot Product)
    v₁ · v₂ = (0.7 × 0.5) + (0.2 × 0.4) + (0.1 × 0.1)
           = 0.35 + 0.08 + 0.01
           = 0.44

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Step 2: 벡터 크기 (L2 Norm)
    ||v₁|| = √(0.7² + 0.2² + 0.1²)
          = √(0.49 + 0.04 + 0.01)
          = √0.54
          = 0.735

    ||v₂|| = √(0.5² + 0.4² + 0.1²)
          = √(0.25 + 0.16 + 0.01)
          = √0.42
          = 0.648

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Step 3: 코사인 유사도
    cos θ = 0.44 / (0.735 × 0.648)
         = 0.44 / 0.476
         = 0.924

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Step 4: 변화량 계산
    change = 1 - 0.924 = 0.076

    판정 (임계값 0.1):
      0.076 < 0.1 → 유의미한 변화 없음 ✓

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

해석:
  • similarity = 0.924 (92.4%)
    → 두 성향 벡터가 매우 유사
  
  • 힐링: 0.7 → 0.5 (감소)
  • 액티비티: 0.2 → 0.4 (증가)
  
  → 방향은 비슷하지만 미세한 변화 발생
  → 아직 큰 변화는 아님 (임계값 미만)
```


### 3.3 Haversine 공식 (구면 거리 계산)

```
┌─────────────────────────────────────────────────────────────┐
│                    Haversine 공식 상세                       │
└─────────────────────────────────────────────────────────────┘

목적:
  지구 표면의 두 지점 간 최단 거리 계산
  (구면 기하학 기반)

공식:
  d = 2R × arcsin(√a)

  여기서:
    a = sin²(Δφ/2) + cos(φ₁) × cos(φ₂) × sin²(Δλ/2)
    
    R = 6371 km (지구 평균 반지름)
    φ₁, φ₂ = 위도 (라디안)
    λ₁, λ₂ = 경도 (라디안)
    Δφ = φ₂ - φ₁
    Δλ = λ₂ - λ₁

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

실제 계산 예시:

  장소 A: 해운대 해변
    위도: 35.1585° N
    경도: 129.1603° E

  장소 B: 광안리 해변
    위도: 35.1532° N
    경도: 129.1189° E

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Step 1: 도 → 라디안 변환
    φ₁ = 35.1585° × π/180 = 0.6138 rad
    φ₂ = 35.1532° × π/180 = 0.6137 rad
    λ₁ = 129.1603° × π/180 = 2.2541 rad
    λ₂ = 129.1189° × π/180 = 2.2534 rad

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Step 2: 차이 계산
    Δφ = 0.6137 - 0.6138 = -0.0001 rad
    Δλ = 2.2534 - 2.2541 = -0.0007 rad

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Step 3: a 계산
    sin²(Δφ/2) = sin²(-0.00005)²
               ≈ 0.0000000025

    cos(φ₁) = cos(0.6138) = 0.8192
    cos(φ₂) = cos(0.6137) = 0.8192

    sin²(Δλ/2) = sin²(-0.00035)²
               ≈ 0.000000122

    a = 0.0000000025 + (0.8192 × 0.8192 × 0.000000122)
      = 0.0000000025 + 0.0000000819
      = 0.0000000844

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Step 4: 거리 계산
    c = 2 × arcsin(√0.0000000844)
      = 2 × arcsin(0.0002906)
      = 2 × 0.0002906
      = 0.0005812 rad

    d = 6371 × 0.0005812
      = 3.70 km

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  결과:
    해운대 → 광안리: 약 3.7 km

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

왜 Haversine을 사용하는가?

  ✗ 유클리드 거리 (직선 거리):
    d = √[(x₂-x₁)² + (y₂-y₁)²]
    → 평면 지도에서만 정확
    → 지구의 곡률 무시
    → 큰 거리에서 오차 발생

  ✓ Haversine (구면 거리):
    → 지구가 구형임을 고려
    → 실제 이동 거리에 근접
    → 항공/해양 내비게이션에서 표준
```

```
            m = 특성 수 (13개)

  STEP 1: 점수 집계
    FOR question IN questions:           // O(n)
      FOR trait IN option.scores:        // O(m)
        scores[trait] += value

    4개 축 계산 (각각 O(1))

  ─────────────────────────────────────────────────────────

  STEP 3: 우선순위 규칙
    8개 규칙 순차 확인 (각각 O(1))
    
    복잡도: O(1)

  STEP 4: 폴백 점수 계산
    8개 유형 점수 계산
    최댓값 찾기
    
    복잡도: O(1)

  ─────────────────────────────────────────────────────────

  전체 시간 복잡도: O(1) - 상수 시간
  공간 복잡도: O(1)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[2] 동적 성향 학습
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

            p = 프로파일 태그 수 (평균 10개)

  STEP 1: 가중치 결정
    복잡도: O(1)

  ─────────────────────────────────────────────────────────

  STEP 2: 프로파일 업데이트
    FOR tag IN event.tags:               // O(k)
      profile[tag] += α × weight × base
    
    복잡도: O(k)
    실제값: O(3) ≈ O(1)

  ─────────────────────────────────────────────────────────

  STEP 3: 벡터 정규화
    sum = 0
    FOR value IN profile.values():       // O(p)
      sum += value
    
    FOR i IN range(p):                   // O(p)
      vector[i] = profile[i] / sum
    
    복잡도: O(p)
    실제값: O(10) ≈ O(1)

  ─────────────────────────────────────────────────────────

  STEP 4: 코사인 유사도
    dot_product = 0
    FOR i IN range(p):                   // O(p)
      dot_product += v1[i] × v2[i]
    
    norm1 = sqrt(sum(v1[i]²))            // O(p)
    norm2 = sqrt(sum(v2[i]²))            // O(p)
    
    similarity = dot / (norm1 × norm2)   // O(1)
    
    복잡도: O(p)

  ─────────────────────────────────────────────────────────

  전체 시간 복잡도: O(p) - 선형 시간
  실제값: O(10) ≈ O(1) - 사실상 상수
  공간 복잡도: O(p)

  실행 시간: < 1ms
```

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Option 1: Greedy Nearest Neighbor (현재 채택)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  복잡도: O(n²)
  정확도: ~80%
  실행 시간: <1ms

  선택 이유:
    ✓ 실시간 응답 필수
    ✓ n이 작음 (평균 3-4개/일)
    ✓ 충분히 좋은 결과
    ✓ 구현 간단

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Option 2: Dynamic Programming
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  복잡도: O(n²2ⁿ)
  정확도: 100%
  실행 시간: ~100ms (n=10)

  미채택 이유:
    ✗ 과도한 계산 비용
    ✗ 실시간 응답 어려움
    ✗ n>15일 때 사용 불가

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Option 3: 2-opt 개선 (향후 고려)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  방법:
    1. Greedy NN으로 초기해 생성
    2. 2-opt로 경로 개선

  복잡도: O(n²) (반복 횟수 제한)
  정확도: ~90%
  실행 시간: ~5ms

  장점:
    ✓ 더 나은 정확도
    ✓ 여전히 빠름
    ✓ 기존 코드 재사용

  고려 시점:
    • 사용자 불만 발생 시
    • 더 긴 일정 지원 시 (n>20)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
실제 영향 분석
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  시나리오: 하루 5개 장소 방문

  최적해:     총 거리 12.0 km, 이동 시간 30분
  Greedy NN:  총 거리 14.5 km, 이동 시간 36분
  
  차이: +2.5 km, +6분
  
  사용자 체감:
    • 이동 시간 차이 미미 (6분)
    • 여유 시간에 흡수됨
    • 불만 발생 가능성 낮음

  결론: Greedy NN 충분
```

## 7. 결론

### 7.1 알고리즘 설계 철학

```
┌─────────────────────────────────────────────────────────────┐
│                  떠나GO AI 설계 원칙                         │
└─────────────────────────────────────────────────────────────┘

1. 실용성 우선 (Pragmatism First)
   ─────────────────────────────
   • 완벽한 알고리즘 < 빠르고 충분히 좋은 알고리즘
   • 이론적 최적 < 사용자 체감 품질
   • O(n²) Greedy NN > O(n²2ⁿ) DP

2. 해석 가능성 (Interpretability)
   ─────────────────────────────
   • 규칙 기반 시스템으로 디버깅 용이
   • 심리학 이론 기반으로 설명 가능
   • 블랙박스 회피

3. 점진적 개선 (Incremental Learning)
   ─────────────────────────────────
   • 규칙 기반으로 시작 → 데이터 수집
   • 동적 학습으로 개인화 향상
   • 향후 ML 모델 전환 준비

4. 실시간 응답 (Real-time Response)
   ────────────────────────────────
   • 모든 알고리즘 <100ms 목표
   • 성향 분석: <1ms
   • 일정 생성: <10ms
   • 사용자 경험 최우선

5. 확장 가능성 (Scalability)
   ──────────────────────────
   • n≤30에서 최적화
   • 향후 n>50 대응 가능 구조
   • 알고리즘 교체 용이한 모듈화
```
